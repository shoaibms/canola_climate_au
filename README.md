# Canola Trial Site Climate Explorer

An interactive Streamlit dashboard for exploring rainfall climatology across canola-growing regions of Australia. Fetches 30+ years of daily weather data from the [SILO DataDrill API](https://www.longpaddock.qld.gov.au/silo/) and visualises monthly rainfall distributions, inter-annual variability, and hierarchical climate clustering across user-defined sites.

![Python](https://img.shields.io/badge/Python-3.9%2B-blue) ![Streamlit](https://img.shields.io/badge/Streamlit-1.x-red) ![License](https://img.shields.io/badge/License-MIT-green)

---

## What it does

- Pulls daily rainfall and temperature data from SILO (no API key required, just an email address)
- Computes monthly climatology statistics: median, standard deviation, P25/P75 per site
- Renders an interactive map of all sites coloured by state
- On site selection: shows monthly rainfall bars with temperature overlay, a rainfall reliability bubble chart, and a hierarchical clustering dendrogram grouping sites by climate similarity

## Setup

**1. Clone the repository and create a virtual environment**

```bash
git clone https://github.com/your-username/canola-climate-explorer.git
cd canola-climate-explorer
python -m venv venv
```

**2. Activate and install dependencies**

```bash
# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate

pip install -r requirements.txt
```

**3. Set your paths**

Open each of the three `.py` scripts and update `DATA_DIR` to point to your local `data/` folder:

```python
# In fetch_silo_data.py, build_site_stats.py, and canola_dashboard.py
DATA_DIR = Path(r"C:/your/path/to/data")   # Windows
DATA_DIR = Path("/your/path/to/data")       # macOS / Linux
```

**4. Set your email**

The SILO API requires a valid email address (no registration, no key). Open `fetch_silo_data.py` and update:

```python
SILO_EMAIL = "your.email@example.com"
```

---

## Running

Run the three scripts in order. Steps 1 and 2 are **one-time setup** — only re-run them if you change your site list or want to refresh the data.

```bash
# Step 1 — fetch SILO climate data (takes a few minutes)
python fetch_silo_data.py

# Step 2 — compute site statistics
python build_site_stats.py

# Step 3 — launch the dashboard
streamlit run canola_dashboard.py
```

## Customising your sites

All site configuration lives in `data/sites_clean.csv`. Open it in any spreadsheet editor or text editor:

```
site_name,site_code,state,lat,lon,environment_group
Wagga Wagga,NW,NSW,-35.119,147.368,Southern NSW
Horsham,VH,VIC,-36.711,142.199,Victoria
Merredin,WMR,WA,-31.482,118.276,Western Australia
```

**To add a site** — append a new row with a unique `site_code`, the town name, state, latitude, longitude, and an environment group label of your choice.

**To remove a site** — delete its row.

**To use completely different locations** — replace the file entirely. The scripts and dashboard will adapt automatically; no code changes are needed.

After any change to `sites_clean.csv`, re-run Steps 1 and 2 before launching the dashboard.

> Latitude and longitude for any Australian location can be found via [the BOM site finder](http://www.bom.gov.au/climate/data/) or Google Maps (right-click → copy coordinates).

## Data source

Weather data is sourced from [SILO DataDrill](https://www.longpaddock.qld.gov.au/silo/), a gridded climate dataset for Australia maintained by the Queensland Government. Data spans 1990–2023 by default; adjust the `start` and `finish` parameters in `fetch_silo_data.py` to change the period.
