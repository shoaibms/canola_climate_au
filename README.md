# Canola Trial Site Climate Explorer

An interactive Streamlit dashboard for exploring rainfall climatology across canola-growing regions of Australia. Fetches 30+ years of daily weather data from the [SILO DataDrill API](https://www.longpaddock.qld.gov.au/silo/) and visualises monthly rainfall distributions, inter-annual variability, and hierarchical climate clustering across user-defined sites.

![Python](https://img.shields.io/badge/Python-3.9%2B-blue) ![Streamlit](https://img.shields.io/badge/Streamlit-1.x-red) ![License](https://img.shields.io/badge/License-MIT-green)

---

## Features

- Pulls daily rainfall and temperature data from SILO (no API key required -- just an email address)
- Computes monthly climatology statistics: median, standard deviation, P25/P75 rainfall, and median Tmax/Tmin per site
- Interactive map of all sites colour-coded by state (NSW, VIC, WA)
- Per-site monthly rainfall bars with temperature overlay
- Rainfall reliability bubble chart (annual median rainfall vs. coefficient of variation)
- Hierarchical clustering dendrogram grouping sites by climate similarity (Ward linkage on 48 features)
- Fully customisable site list -- add, remove, or replace sites without changing any code

## Quick Start

### 1. Clone and install

```bash
git clone https://github.com/your-username/canola-climate-explorer.git
cd canola-climate-explorer
python -m venv venv
```

Activate the virtual environment:

```bash
# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

### 2. Set your email (for SILO API)

The SILO API requires a valid email address (no registration or API key needed). Open `code/fetch_silo_data.py` and update:

```python
SILO_EMAIL = "your.email@example.com"
```

### 3. Run

```bash
# Option A -- use the included data (skip straight to the dashboard)
streamlit run code/canola_dashboard.py

# Option B -- fetch fresh data for your own sites, then launch
python code/fetch_silo_data.py        # Step 1 (takes a few minutes)
python code/build_site_stats.py       # Step 2
streamlit run code/canola_dashboard.py # Step 3
```

## Customising Sites

All site configuration lives in **`data/sites_clean.csv`**. Open it in any spreadsheet editor or text editor:

```csv
site_name,site_code,state,lat,lon,environment_group
Wagga Wagga,NW,NSW,-35.119,147.368,Southern NSW
Horsham,VH,VIC,-36.711,142.199,Victoria
Merredin,WMR,WA,-31.482,118.276,Western Australia
```

| Action | What to do |
|--------|------------|
| **Add a site** | Append a new row with a unique `site_code`, town name, state, latitude, longitude, and an environment group label of your choice. |
| **Remove a site** | Delete its row. |
| **Replace all sites** | Overwrite the file entirely. The scripts and dashboard adapt automatically -- no code changes needed. |

After any change to `sites_clean.csv`, re-run **Steps 1 and 2** before launching the dashboard.

> Latitude and longitude for any Australian location can be found via [Google Maps](https://maps.google.com) (right-click a point and copy coordinates) or the [BOM station finder](http://www.bom.gov.au/climate/data/).

## Data Source

Weather data is sourced from [SILO DataDrill](https://www.longpaddock.qld.gov.au/silo/), a gridded climate dataset for Australia maintained by the Queensland Government. The default period is 1990--2023; adjust the `start` and `finish` parameters in `code/fetch_silo_data.py` to change the date range.
